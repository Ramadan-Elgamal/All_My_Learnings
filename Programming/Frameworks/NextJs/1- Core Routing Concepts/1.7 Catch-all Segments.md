# Catch-all Segments
Keep your editor open. We just conquered dynamic routes, which let us use a [slug] folder to catch a single piece of the URL. But pretty quickly in the real world, you hit a wall where one single placeholder isn't enough.
## The "Why": The Deeply Nested Problem
Imagine you are building a documentation site (like the Next.js docs). Your URLs might look wildly different depending on how deep the user clicks:
 * /docs/installation (1 segment deep)
 * /docs/routing/pages-and-layouts (2 segments deep)
 * /docs/api-reference/components/link/props (4 segments deep)
If we use standard dynamic routing, a app/docs/[slug]/page.tsx file will only catch the first one. To catch the second, we'd need app/docs/[category]/[slug]/page.tsx. To catch the third, we'd need four nested bracket folders.
We don't know in advance how deep the URL will go. We need a way to tell the router, *"Hey, starting from this folder, just grab the rest of the URL, no matter how long it is."*
## The Mental Model: The Fishing Net
A standard dynamic route [slug] is like a single fishing pole. It catches exactly one fish (one URL segment) and stops.
A **Catch-all Segment** is a wide fishing net. You throw it out, and it scoops up absolutely everything remaining in the URL path, bringing it all onto your boat at once.
## Concept Before Syntax: Arrays, Not Strings
Because a catch-all route can capture multiple URL segments, Next.js can't just give you a single string like it does with a normal dynamic route.
Instead, it chops the URL up at every slash (/) and hands you an **array of strings**.
If your route catches /docs/framework/react/hooks, the data Next.js gives your component won't be "framework/react/hooks". It will be ["framework", "react", "hooks"].
## Building Incrementally: The Spread Syntax
To create a catch-all segment, we borrow a concept from JavaScript: the spread operator (...). You create a folder with square brackets, but you add three dots before the name: [...slug].
Let's look at the code.
```tsx
// File: app/docs/[...slug]/page.tsx

export default function DocumentationPage({ params }: { params: { slug: string[] } }) {
  // Notice that params.slug is now an Array of strings!
  
  return (
    <main>
      <h1>Viewing Documentation Path:</h1>
      {/* We can join the array back together with arrows for a breadcrumb effect */}
      <p>{params.slug.join(' -> ')}</p>
    </main>
  );
}

```
If a user visits /docs/getting-started/setup, the screen will render: getting-started -> setup.
## Teaching Patterns: The "Optional" Catch-all
There is a slight catch to the standard catch-all. A folder named [...slug] requires *at least one* segment to match.
If you have app/docs/[...slug]/page.tsx, it perfectly handles /docs/react and /docs/react/hooks. But if the user navigates to the plain root /docs, it will throw a 404 Error! The net expects to catch at least one fish.
Usually, you want that root /docs page to show your main documentation hub. You *could* create a separate app/docs/page.tsx file next to your catch-all folder to handle the root. But Next.js gives us a cleaner pattern: **Optional Catch-all Segments**.
You wrap the folder in *double* square brackets: [[...slug]].
This tells Next.js: *"Catch all the segments if they are there, but if the user just goes to the root folder, that's completely fine too. Give me an empty array."* ## Mistakes to Avoid Before You Make Them
 * **Mistake 1: Treating a catch-all parameter as a string.**
   * *Reality:* If you try to run standard string methods like params.slug.toUpperCase() on a catch-all route, your app will crash. It is an array. You must loop over it (.map()) or join it (.join()) first.
 * **Mistake 2: Putting a catch-all in the middle of a path.**
   * *Reality:* You cannot create a folder structure like app/shop/[...categories]/checkout/page.tsx. A catch-all net sweeps up *everything* to the end of the URL. It must be the absolute last folder in your routing tree (the leaf node).
 * **Mistake 3: Getting confused by [...slug] vs [[...slug]].**
   * *Reality:* Single brackets = Requires at least one segment. Double brackets = Can be empty (optional). When in doubt for things like dashboards or docs, default to the double brackets.
## Practical Exercise
Let's build an optional catch-all route to see how the array behaves.
 1. In your app directory, create a folder named docs.
 2. Inside docs, create an optional catch-all folder. Literally type: [[...topic]].
 3. Inside [[...topic]], create your page.tsx file.
 4. Add this code to inspect the array:
   ```tsx
   export default function DocsPage({ params }: { params: { topic?: string[] } }) {
     // Because it's optional, params.topic might be undefined at the root.
     // We safely fall back to an empty array.
     const pathArray = params.topic || [];
   
     if (pathArray.length === 0) {
       return <h1>Welcome to the Docs Homepage</h1>;
     }
   
     return (
       <div>
         <h1>Deep Dive into: {pathArray[pathArray.length - 1]}</h1>
         <p>Full path length: {pathArray.length} segments deep.</p>
       </div>
     );
   }
   
   ```
 5. Open your browser and go to http://localhost:3000/docs. You will see the Homepage message.
 6. Now go to http://localhost:3000/docs/react/hooks/use-state.
 7. You will see it successfully parsed the URL, realizing you are 3 segments deep, and isolated "use-state" as the current topic. One file, infinite URLs.
