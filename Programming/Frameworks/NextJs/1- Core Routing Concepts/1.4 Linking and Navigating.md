# Linking and Navigating

We now have multiple pages organized into folders, with layouts wrapping them correctly. The remaining problem is navigation. At the moment, the only way to move between pages is by manually typing URLs into the browser's address bar.

To create a proper user experience, we need client-side navigation.

---

# Why We Don't Use Regular `<a>` Tags

In traditional websites, clicking an HTML `<a>` tag tells the browser to completely reload the page.

The browser will:

1. Destroy the current page.
2. Request a new HTML document from the server.
3. Download that document.
4. Rebuild the entire page from scratch.

This causes:

- Full page refreshes
- Loss of React state
- Layouts being recreated
- Flashing or white loading screens

If you use a normal `<a>` tag inside a Next.js application, this is exactly what happens.

What we actually want is a **client-side transition**.

Instead of rebuilding the entire page, React should simply replace the page content while keeping everything else—such as layouts—intact.

---

# The `<Link>` Component

Next.js provides the `<Link>` component as a replacement for the HTML `<a>` tag.

Instead of asking the browser to load a new document, `<Link>` intercepts the click event and tells Next.js to:

- Fetch only the code for the destination page.
- Load any required route data.
- Replace the current page inside the existing layout.

This creates fast, seamless navigation.

```tsx
import Link from "next/link";

export default function Navbar() {
  return (
    <nav>
      {/* Recommended */}
      <Link href="/about">About Us</Link>

      {/* Avoid */}
      <a href="/contact">Contact</a>
    </nav>
  );
}
```

---

# What Happens Behind the Scenes?

When you click a `<Link>`:

```
Current Layout
 ┌─────────────────────────┐
 │ Navbar                  │
 │ Sidebar                 │
 │                         │
 │ Current Page            │
 │        ↓                │
 │ New Page                │
 └─────────────────────────┘
```

Only the page content changes.

The layout stays mounted.

This means:

- Faster navigation
- State is preserved
- No flashing screen
- Better user experience

---

# Prefetching

One of the biggest advantages of `<Link>` is **Prefetching**.

When a `<Link>` becomes visible in the viewport, Next.js automatically begins downloading the code for its destination in the background.

By the time the user clicks the link, much—or all—of the required code has already been downloaded.

As a result, navigation feels almost instantaneous.

### Restaurant Analogy

Imagine you're reading a dessert menu.

Before you order, the waiter notices your interest and asks the chef to prepare the cheesecake.

When you finally say:

> "I'll have the cheesecake."

It's already ready and arrives immediately.

Next.js does the same thing with routes.

# Programmatic Navigation with `useRouter`

Sometimes navigation should happen because of logic rather than because the user clicked a link.

Examples include:

- Successful login
- Successful signup
- Completing a payment
- Finishing a form
- Timer expiration

In these situations, use the `useRouter` hook.

```tsx
"use client";

import { useRouter } from "next/navigation";

export default function LoginForm() {
  const router = useRouter();

  const handleSubmit = (e) => {
    e.preventDefault();

    // Authentication logic...

    router.push("/dashboard");
  };

  return (
    <form onSubmit={handleSubmit}>
      <button type="submit">
        Log In
      </button>
    </form>
  );
}
```

# Common Mistakes

## 1. Importing `useRouter` from the Wrong Package

### Incorrect

```tsx
import { useRouter } from "next/router";
```

This is the Pages Router API.

### Correct

```tsx
import { useRouter } from "next/navigation";
```

For the App Router, always import from:

```text
next/navigation
```

---

## 2. Wrapping `<a>` Inside `<Link>`

Older Next.js versions required:

```tsx
<Link href="/about">
  <a>About</a>
</Link>
```

This is no longer necessary.

Simply write:

```tsx
<Link href="/about">
  About
</Link>
```

`<Link>` already renders an anchor element internally.

---

## 3. Forgetting `"use client"`

`useRouter` is a React hook.

Hooks only run in Client Components.

Always place this directive at the top of the file:

```tsx
"use client";
```

Without it, Next.js will throw an error.
