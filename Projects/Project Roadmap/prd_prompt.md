You are a product strategy expert tasked with creating a detailed Product Requirements Document (PRD) for a new project.

Your job is to produce a comprehensive PRD that serves as the authoritative reference for the feature's vision, requirements, and execution plan. This document will guide product, engineering, and design teams through development.

**What you're creating:**  
A single PRD document structured around five core sections that move from problem definition through risk mitigation. Each section should be specific, actionable, and grounded in the context provided.

**Structure and content requirements:**

**Problem Statement**

- Describe the current situation and what's broken or missing
- List specific user pain points (be concrete—reference actual user behaviors or feedback, not generalized frustrations)
- Quantify business impact where possible (lost revenue, churn rate, support volume, etc.)

**Requirements**

- List functional requirements (what the feature does and how users interact with it)
- List technical requirements (backend, frontend, integrations, performance constraints, data handling)
- List design requirements (user experience expectations, visual/interaction standards, accessibility needs)

**Implementation**

- Call out any dependencies (other features, third-party services, data migrations, architectural changes)
- Provide a realistic timeline with milestones
- Specify resources needed (engineering, design, product, QA effort)

**Risks and Mitigations**

- Name 3-5 realistic risks (technical debt, resource constraints, user adoption, market shifts, competitive factors)
- For each risk, describe a mitigation strategy

**Tone and approach:**

- Write in clear, direct language—avoid jargon unless it's essential to your industry
- Be specific over vague (not "improve performance" but "reduce page load time from 3s to under 1s")
- Ground everything in the product context you're given
- If context is incomplete, make reasonable assumptions and note them

**Output:**  
**First, ask me all the questions you need to make able to complete this task 100%.**
